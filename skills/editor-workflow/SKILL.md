# Editor Workflow

## Use This Skill For

- section navigation
- form interaction design
- progress display
- incomplete state handling
- section-specific editing behavior

## Responsibilities

- Show one active section at a time.
- Keep the user moving even with partial information.
- Make the checklist feel structured, not overwhelming.

## Rules

- Never require all fields before navigation.
- Prefer small focused forms over large mixed panels.
- Preserve stable section ordering.
- Compute progress from meaningful completion signals, not page visits alone.

## UX Checks

- Can a user start from a rough requirement immediately?
- Does the UI reduce cognitive overload?
- Is it clear what section the user is editing now?
- Can the user see output improvements as they refine answers?
