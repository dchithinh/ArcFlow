# UI System

## Use This Skill For

- layout system
- reusable form primitives
- preview presentation
- responsive behavior
- visual design consistency

## Responsibilities

- Support the three-pane mental model where screen size allows it.
- Keep the editing experience clear and technical rather than decorative.
- Make dense firmware design information readable.

## Rules

- Favor clarity over novelty in form controls.
- Keep navigation, editing, and preview visually distinct.
- Preserve usability on smaller screens with stacked layouts.
- Do not let visual styling obscure completion state or data relationships.

## Review Checks

- Is the active section obvious?
- Is the preview readable without editing fatigue?
- Does the layout degrade cleanly on mobile?
- Are form controls consistent across sections?
